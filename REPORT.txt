3. Image Size and How I Kept It Small
I checked my image sizes using:

==Code==
  
docker images
Docker tagged my app image as :latest by default.
•	213MB on disk
•	51.4MB actual content size
This is normal for a small Python/FastAPI app.
  -----------------------------------------------------

The Ollama image is much larger (3.79GB) because it includes:
•	The model runtime
•	The downloaded model weights
This is expected and matches the assignment instructions.
  -----------------------------------------------------

To keep my app image small, I:
•	Used a slim Python base image
•	Installed only the packages I needed
•	Ordered Docker layers correctly
•	Kept model weights in a volume instead of inside the image

5. Model Persistence
I tested that the model stays saved even if the container is removed.

I ran:

==Code== 

docker-compose down
docker rm project7-sa-ollama-1
docker-compose up -d

Then I checked the logs:

==Code==

docker logs project7-sa-ollama-1

The model did not re-download.
There was no “pulling model…” message in the logs.
This confirms that the named volume ollama-models is working correctly and that it stores the model between container restarts.
